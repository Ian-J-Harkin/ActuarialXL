- The fallback path traversal logic in `Program.cs` goes exactly four parent folders up from the base directory, creating a highly fragile runtime path resolution that will fail when the application is published, containerized, or run from any directory layout other than standard debug output.
- The runtime configuration is coupled to a documentation markdown file (`docs/governance/master-prompt-engineering-log.md`). Runtimes should not depend on files in documentation folders, which are subject to format modifications, linting, and manual edits that will cause unexpected runtime failure.
- The regex used to parse the system prompt is fragile and assumes a rigid block format (`### The System Prompt\s*```text\s*(.*?)\s*````). Any minor change in the markdown, such as changing the code block type to `markdown`, adjusting the header casing, or introducing a trailing space, will result in an unhandled parsing crash.
- Blocking synchronous file system read (`System.IO.File.ReadAllText`) is executed directly inside the dependency injection configuration lambda. This blocks the main thread during startup and can execute multiple times depending on container validation routines.
- There is a complete lack of logging or friendly diagnostic messages when the prompt file resolution or regex matching fails, throwing raw, unhandled exceptions that crash the process without explaining the root cause to the operator.
- The C# compilation loop in `ReconciliationOrchestrator.ProcessBlockAsync` is an all-or-nothing operation; if a single partition fails compilation after 3 retries, an exception is thrown, discarding all successfully compiled outputs that were accumulated in the local `results` list prior to the failure.
- Arbitrary renaming of properties on core models (`CleanHeaderName` to `ExtractedHeaderName`, `Rows` to `DataRows`, and `RowNumber` to `RowIndex`) breaks existing serialization contracts, database schemas, and JSON files without providing backward compatibility or migration mappings.
- Removing the primary constructor from `ColumnDefinition` in favor of `{ get; init; }` properties breaks compile-time compatibility with any caller in the codebase that instantiates the class using parameter passing.
- Renaming `RowNumber` to `RowIndex` introduces index semantic ambiguity, blurring the distinction between Excel's 1-based layout conventions and code-level 0-based arrays, increasing the risk of off-by-one errors in spreadsheet processing.
- The unit test project is polluted with integration testing dependencies (`Microsoft.AspNetCore.Mvc.Testing` and `Microsoft.EntityFrameworkCore.Sqlite`) and references to API/Persistence projects, violating clean separation of test suites and slowing down unit test execution.
- The `IDomainInterrogationBridge` interface leaks compiler concerns by accepting `string? previousCompilerError` directly in its contract, coupling the domain interrogation abstraction to Roslyn compiler implementation details.
- Interface contract modifications to `IDomainInterrogationBridge` and `IReconciliationOrchestrator` break third-party implementers, test mocks, or alternative engines, yet the corresponding code fixes for those implementations are entirely absent from the diff.
