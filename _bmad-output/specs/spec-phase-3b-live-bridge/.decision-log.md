## Architectural Decisions

### DECISION-001: Configuration-Driven AST Safety Scanner
**Context:** The `AstSafetyScanner` requires a strict list of allowed and forbidden namespaces to prevent malicious or accidental system access by the dynamically compiled LLM code. Originally, these lists were hardcoded.
**Decision:** Shifted the `AllowedNamespaces` and `ForbiddenNamespaces` from hardcoded hash sets in the `AstSafetyScanner` class to the `LlmBridgeConfiguration` object (`appsettings.json`).
**Rationale:** A hardcoded deny/allow list creates excessive maintenance burden. If the LLM legitimately requires a new standard library namespace in the future, it would require a code change and a full system redeploy. Moving this to configuration allows a system administrator to manage the security perimeter via `appsettings.json` without recompilation, paving the way for a future Admin UI in Phase V.
**Status:** Implemented (Passed 15/15 unit tests)
